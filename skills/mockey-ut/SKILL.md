# mockey Unit Test Mock Guide

mockey (`github.com/bytedance/mockey`) is a Go runtime mock library that replaces functions and variables by rewriting function instructions. No need to declare all dependencies as interfaces for injection.

> **Prerequisite**: Inlining and compiler optimizations must be disabled. Use `go test -gcflags="all=-l -N"`.

## When to Use

- Unit tests need to mock external functions, methods, or variables when interface injection is inconvenient or impossible
- Need to return different values based on conditions (conditional mocking) or in sequence (sequence mocking)
- Need to execute original logic alongside mocking (decorator pattern)

## Mock Strategy Priority

When choosing a mock approach, follow this priority:

1. **Library/framework-specific mock capabilities** — if the dependent library or type provides mock interfaces or test utilities (e.g., `httptest`, `redismock`), use those first.
2. **mockey as fallback** — when the dependency has no mock capability, or interface injection is inconvenient, use mockey for runtime patching.

See `references/api-reference.md` for the complete API reference.

## Lifecycle Management

mockey provides two functions for automatic mock lifecycle management. Mocks within the scope are automatically released when the scope ends:

### PatchConvey

Wraps `goconvey.Convey`, auto-releases mocks at the end of the Convey block. **Use when the project already uses the goconvey framework**, replacing existing `Convey` calls.

```go
import (
    "testing"
    "github.com/bytedance/mockey"
    c "github.com/smartystreets/goconvey/convey"
)

func TestExample(t *testing.T) {
    mockey.PatchConvey("description", t, func() {
        mockey.Mock(targetFunc).Return(mockValue).Build()
        // mock is active here
        c.So(result, c.ShouldEqual, expected)
    })
    // mock is auto-released
}
```

### PatchRun

Lightweight scope, **no goconvey dependency**. Auto-releases mocks when the function ends.

```go
func TestExample(t *testing.T) {
    mockey.PatchRun(func() {
        mockey.Mock(targetFunc).Return(mockValue).Build()
        // mock is active here
    })
    // mock is auto-released
}
```

### Selection Guide

| Scenario | Recommendation |
| :--- | :--- |
| Project already uses goconvey (`Convey` + `So` assertions) | `PatchConvey` |
| Project uses standard testing or other assertion libraries | `PatchRun` |
| Need manual control over mock lifecycle | `Build()` + `defer mocker.UnPatch()` |

Both support nesting — each level only releases its own mocks.

### Manual Release

If not using `PatchConvey` / `PatchRun`, mockey patches are **not auto-released** and must be manually `UnPatch`ed:

```go
mocker := mockey.Mock(targetFunc).Return(mockValue).Build()
defer mocker.UnPatch()
```

**Regardless of whether mockey lifecycle functions are used, mocks/patches created by non-mockey libraries (e.g., other mock libraries, manually replaced variables) must be released by the caller.**

## Build Flags

mockey requires inlining and compiler optimizations to be disabled:

```bash
go test -gcflags="all=-l -N" ./...
```

- **Goland**: Add `-gcflags="all=-l -N"` in Run/Debug Configurations > Go tool arguments
- **VSCode**: Add `"go.buildFlags": ["-gcflags=all=-N -l"]` in `settings.json`

## Code Style

**Dot imports are forbidden**. mockey's own README heavily uses `import . "github.com/bytedance/mockey"`, but code generated by this skill always uses the `mockey.` prefix and alias `c` for goconvey:

```go
import (
    "github.com/bytedance/mockey"
    c "github.com/smartystreets/goconvey/convey"
)
```

This is the default style for this skill. If the user has explicit requirements, other skills have explicit constraints, or the repo's existing tests have established conventions, follow those instead.

## Troubleshooting

Quick reference for common issues. See `references/troubleshooting.md` for details:

| Symptom | Common Cause |
| :--- | :--- |
| Mock not taking effect | Missing `-gcflags="all=-l -N"`; forgot `Build()`; signature mismatch |
| "re-mock" panic | Function mocked twice in the same PatchConvey/PatchRun |
| "function is too short to patch" | Function too short or inlining not disabled; try `MockUnsafe` |
| "signal SIGBUS" (Apple Silicon Mac) | Occasional arm64 issue; retry or upgrade to latest |

Full troubleshooting guide in `references/troubleshooting.md`.
