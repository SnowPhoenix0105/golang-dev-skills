# Fyne 排障思路

## 1. Widget 不可见 / 不渲染

**最常见的问题**。

### 检查清单

1. **`CreateRenderer()` 是否返回了 nil？**
   ```go
   func (w *MyWidget) CreateRenderer() fyne.WidgetRenderer {
       return nil  // ❌ 这是原因
   }
   ```

2. **`Objects()` 是否返回空切片或含 nil 元素？**
   ```go
   // ❌ 空切片
   func (r *myRenderer) Objects() []fyne.CanvasObject {
       return []fyne.CanvasObject{}
   }

   // ✅ 至少返回一个有效对象
   func (r *myRenderer) Objects() []fyne.CanvasObject {
       return []fyne.CanvasObject{r.bg}
   }
   ```

3. **是否调用了 `ExtendBaseWidget()`？**
   ```go
   // ❌ 忘记调用
   func NewMyWidget() *MyWidget {
       return &MyWidget{}
   }

   // ✅ 必须调用
   func NewMyWidget() *MyWidget {
       w := &MyWidget{}
       w.ExtendBaseWidget(w)
       return w
   }
   ```

4. **`canvas.Line` 等类型使用了值接收者？**
   ```go
   // ❌ 值类型不满足 CanvasObject 接口
   line := canvas.Line{}

   // ✅ 使用指针
   line := &canvas.Line{}
   ```

5. **`Resize()` 是否给 Widget 分配了尺寸？**
   无布局容器中的 Widget 默认尺寸为 0，必须手动 Resize：
   ```go
   btn := widget.NewButton("Click", fn)
   btn.Resize(fyne.NewSize(200, 40))  // 必须设定大小才能点击
   ```

## 2. 点击无响应 / Tappable 不触发

### 常见原因

1. **Widget 尺寸为 0×0**：在没有 Layout 的容器中，需要手动 `Resize()`
2. **没有实现 `Tappable` 接口**：确认 Widget 实现了 `Tapped(*PointEvent)` 方法
3. **Widget 被更上层的 Widget 遮挡**：检查 OverlayStack 或 Stack 中的对象顺序
4. **DisableableWidget 被禁用了**：检查 `Disabled()` 返回值

## 3. panic: interface conversion: Layout is nil

```
panic: interface conversion: fyne.Layout is nil, not *widget.ListLayout
```

### 原因
在 Widget 的 Renderer 创建之前调用了需要 Layout 的方法（如 `Resize()`）。

### 解决方案
```go
// ❌ 不要在 Widget 创建后立即调用 Resize
list := widget.NewList(lenFn, createFn, updateFn)
list.Resize(fyne.NewSize(100, 100))  // 可能 panic

// ✅ 放入容器后再 Resize 容器
list := widget.NewList(lenFn, createFn, updateFn)
c := container.NewVBox(list)
c.Resize(fyne.NewSize(100, 100))
```

## 4. panic: fyne.Canvas is nil

```
panic: interface conversion: fyne.Canvas is nil
```

### 原因
在没有加入 Window 的情况下调用了需要 Canvas 的方法。

### 解决方案
- 将 Widget 加入 Window/Canvas 后再进行操作
- 在测试中使用 `test.NewCanvas()` 提供 Canvas 上下文

## 5. 数据绑定不更新

### 检查清单

1. **使用的是 `Set()` 还是直接赋值？**
   ```go
   data := binding.NewString()
   // ✅ 会触发通知
   data.Set("new value")
   // ❌ 不会触发通知（如果 data 是外部绑定的变量）
   myVar = "new value"  // 需调用 data.Reload()
   ```

2. **外部绑定需要手动 Reload**
   ```go
   var count int
   b := binding.BindInt(&count)
   count = 10        // 直接修改不会通知
   b.Reload()        // 需要显式调用 Reload
   ```

3. **监听器是否被正确注册？**
   ```go
   data.AddListener(binding.NewDataListener(func() {
       // 验证这个回调是否被执行
       fmt.Println("data changed")
   }))
   ```

4. **Widget 是否使用了 WithData 构造函数？**
   ```go
   // ✅ 自动绑定
   label := widget.NewLabelWithData(data)

   // ❌ 不会自动更新
   label := widget.NewLabel(data.Get())  // 不存在 Get 方法，这只是示意
   ```

## 6. 布局问题

### 6.1 子元素重叠或被截断

- 检查父容器是否足够大：`parent.Resize(newSize)`
- `MinSize` 返回值是否过小：子 Widget 的 MinSize 应大于等于实际需要

### 6.2 Grid 布局中出现空白格

- 隐藏的对象仍然占据格子位置。如果需要跳过隐藏对象，应该从容器 Objects 中移除，而非只设置 `Hidden = true`

### 6.3 Form 布局第二列（canvas.Text）显示异常

- `layout.FormLayout` 对 `canvas.Text` 处理不佳，第二列的 `canvas.Text` 应在 `Layout()` 中被正确设置宽度

## 7. 资源问题

### 7.1 fyne package / fyne release 后资源加载失败

- 使用 `fyne bundle` 或 `//go:generate fyne bundle` 将资源嵌入代码
- 打包后文件系统路径不可用，必须用嵌入资源

### 7.2 自定义字体不生效

```go
// ✅ 使用嵌入的字体资源
font := &canvas.Text{Text: "Hello", TextStyle: fyne.TextStyle{...}}

// ✅ 加载字体
canvas.NewText("Hello", color.White).TextStyle = fyne.TextStyle{
    Bold:      true,
    Italic:    false,
    Monospace: false,
}
```

## 8. 线程相关问题

### 8.1 从后台 goroutine 直接操作 UI 导致竞态

**症状**：随机 panic、数据竞争、UI 不更新或显示异常

**解决**：
```go
// ❌
go func() {
    widget.SetText("updated")  // 竞态风险
}()

// ✅
go func() {
    result := work()
    fyne.Do(func() {
        widget.SetText(result)
    })
}()
```

### 8.2 v2.6.0+ 迁移相关

- 在 `FyneApp.toml` 中设置 `fyneDo = true`
- 移除自定义 Widget 中的 mutex（框架已通过单 goroutine 保证安全）
- 不再使用 `BaseWidget.SetFieldsAndRefresh`（已移除）

### 8.3 死锁

**常见场景**：
- 在 `FocusGained` / `FocusLost` 中操作 OverlayStack 或 Focus
- 在 `Layout()` 中调用 `Refresh()`
- 在 `fyne.DoAndWait` 中再次调用 `fyne.DoAndWait`

**解决**：避免在回调中改变画布状态（overlays 和 focus），将这类操作推迟到事件处理完成后。

## 9. 调试技巧

### 9.1 启用 Debug 渲染

```bash
go run -tags debug main.go
```
会显示 Fyne 的调试边框，帮助理解布局和碰撞检测区域。

### 9.2 使用 LogError

```go
fyne.LogError("reason", err)  // 输出到控制台，包含文件和行号
```

### 9.3 使用 RenderToMarkup 调试

```go
markup := test.RenderObjectToMarkup(myWidget)
fmt.Println(markup)  // 查看 Widget 的结构
```

### 9.4 检查主题问题

```go
// 验证当前主题配置
settings := fyne.CurrentApp().Settings()
t := settings.Theme()
fmt.Printf("Variant: %v, Scale: %.2f\n", settings.ThemeVariant(), settings.Scale())

// 重置为默认主题
settings.SetTheme(theme.DefaultTheme())
```

### 9.5 验证 WidgetRenderer 状态

```go
r := test.WidgetRenderer(myWidget)
for i, obj := range r.Objects() {
    fmt.Printf("Object[%d]: type=%T, pos=%v, size=%v, visible=%v\n",
        i, obj, obj.Position(), obj.Size(), obj.Visible())
}
```

## 10. 常见错误消息速查

| 错误信息 | 常见原因 | 解决 |
|---------|---------|------|
| `interface conversion: fyne.Layout is nil` | 在 Renderer 创建前调用了 Resize | 先放入 Canvas/Window 再操作 |
| `interface conversion: fyne.Canvas is nil` | Widget 不在窗口层级中 | 使用 test.NewCanvas() 或先 Show() |
| `fyne.WidgetRenderer is nil` | CreateRenderer 返回 nil | 确保返回有效的 Renderer |
| `preferences requires a unique ID` | 未使用 app.NewWithID() | 使用 NewWithID 或设置 FyneApp.toml 的 ID |
| Widget 可见但不可点击 | 尺寸为 0 或未实现 Tappable | Resize 或实现 Tapped 方法 |
| Layout recursively calls Refresh | Layout 中调用了 Refresh | 不要在 Layout 中调用 Refresh |
