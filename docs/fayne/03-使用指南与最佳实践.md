# Fyne 使用指南与最佳实践

## 1. 项目结构

```
myapp/
├── main.go              # 入口，app 初始化，启动
├── FyneApp.toml         # 元数据
├── ui/
│   ├── views/           # 页面/视图
│   ├── widgets/         # 自定义组件
│   └── theme/           # 自定义主题
├── model/               # 数据模型
└── bindings/            # 响应式绑定
```

### FyneApp.toml 示例

```toml
[Details]
  ID = "com.example.myapp"
  Name = "My Application"
  Version = "1.0.0"
  Build = 1
  Icon = "Icon.png"

[Migrations]
  fyneDo = true  # v2.6.0+ 线程模型
```

## 2. 最小化应用模板

```go
package main

import (
    "fyne.io/fyne/v2/app"
    "fyne.io/fyne/v2/container"
    "fyne.io/fyne/v2/widget"
)

func main() {
    a := app.New()
    w := a.NewWindow("Hello")

    hello := widget.NewLabel("Hello Fyne!")
    w.SetContent(container.NewVBox(
        hello,
        widget.NewButton("Hi!", func() {
            hello.SetText("Welcome :)")
        }),
    ))

    w.ShowAndRun()  // 阻塞，等同于 w.Show(); a.Run()
}
```

## 3. Layout 最佳实践

### 3.1 使用容器组合而非绝对定位

```go
// ✅ 推荐：声明式布局
content := container.NewBorder(
    header,                         // top
    footer,                         // bottom
    sidebar,                        // left
    nil,                            // right
    mainContent,                    // center
)

// ✅ 推荐：嵌套组合
form := container.NewVBox(
    container.NewHBox(widget.NewLabel("Name:"), nameEntry),
    container.NewHBox(widget.NewLabel("Age:"), ageEntry),
    widget.NewButton("Submit", onSubmit),
)

// ❌ 避免：绝对定位
btn.Move(fyne.NewPos(100, 200))
```

### 3.2 Border 布局参数说明

```go
container.NewBorder(top, bottom, left, right, center...)
```
- top/bottom：固定 MinSize 高度，宽度自适应
- left/right：固定 MinSize 宽度，高度自适应
- center (objects[0])：占据剩余空间
- objects[1:]：堆叠在中心区域上方（Stack 模式）

nil 表示该边缘不放置内容。

### 3.3 使用 Spacer 实现弹性间距

```go
import "fyne.io/fyne/v2/layout"

// 让按钮靠右显示
toolbar := container.NewHBox(
    layout.NewSpacer(),  // 占据所有剩余空间
    widget.NewButton("OK", fn),
)
```

### 3.4 利用 GridWrap 实现响应式列表

```go
// 每个 item 固定大小，超出宽度自动换行
grid := container.NewGridWrap(
    fyne.NewSize(120, 80),
    items...,
)
```

## 4. 数据绑定最佳实践

### 4.1 首选数据绑定而非手动同步

```go
// ✅ 响应式：数据变更 → UI 自动更新
data := binding.NewString()
data.Set("Hello")

entry := widget.NewEntryWithData(data)
label := widget.NewLabelWithData(data)

// 用户在 Entry 输入 → data 自动更新 → Label 自动更新
// 代码设置 data.Set(...) → Entry 和 Label 都自动更新

// ❌ 避免：手动同步，容易遗漏
label.SetText(entry.Text)
```

### 4.2 集合绑定与 List

```go
items := binding.NewStringList()
items.Set([]string{"Alice", "Bob", "Charlie"})

list := widget.NewListWithData(items,
    func() fyne.CanvasObject {
        return widget.NewLabel("template")
    },
    func(item binding.DataItem, obj fyne.CanvasObject) {
        obj.(*widget.Label).Bind(item.(binding.String))
    })
```

### 4.3 外部绑定与变量同步

```go
var config AppConfig
// 当 config 的值被外部代码修改时，所有绑定的 Widget 自动更新
nameBinding := binding.BindString(&config.Name)
```

### 4.4 转换与格式化

```go
// Int ↔ String 转换
binding.IntToString(myIntBinding)     // Int -> String
binding.StringToInt(myStrBinding)     // String -> Int

// 格式化
binding.FormatFloat(myFloatBinding, "%.2f")
binding.FormatInt(myIntBinding, "%d items")

// sprintf（通用格式化）
binding.SPrintf("%v / %v", currentBinding, totalBinding)
```

## 5. 自定义 Widget 开发

### 5.1 完整实现模板

```go
package mywidget

import (
    "fyne.io/fyne/v2"
    "fyne.io/fyne/v2/canvas"
    "fyne.io/fyne/v2/theme"
    "fyne.io/fyne/v2/widget"
)

// 声明式对象 — 持有 widget 的状态
type MyWidget struct {
    widget.BaseWidget

    Text  string
    Color color.Color

    OnTapped func()
}

func NewMyWidget(text string, tapped func()) *MyWidget {
    w := &MyWidget{Text: text, OnTapped: tapped}
    w.ExtendBaseWidget(w)  // 必须调用
    return w
}

func (w *MyWidget) CreateRenderer() fyne.WidgetRenderer {
    bg := canvas.NewRectangle(theme.Color(theme.ColorNameButton))
    label := canvas.NewText(w.Text, theme.Color(theme.ColorNameForeground))
    return &myRenderer{
        bg:    bg,
        label: label,
        w:     w,
        objects: []fyne.CanvasObject{bg, label},
    }
}

func (w *MyWidget) SetText(text string) {
    w.Text = text
    w.Refresh()
}

// 实现 Tappable 接口
func (w *MyWidget) Tapped(ev *fyne.PointEvent) {
    if w.OnTapped != nil {
        w.OnTapped()
    }
}

// 渲染器
type myRenderer struct {
    bg, label fyne.CanvasObject
    w         *MyWidget
    objects   []fyne.CanvasObject
}

func (r *myRenderer) Destroy() {}

func (r *myRenderer) Layout(size fyne.Size) {
    r.bg.Resize(size)
    r.label.Move(fyne.NewPos(4, 4))
}

func (r *myRenderer) MinSize() fyne.Size {
    return fyne.NewSize(r.label.MinSize().Width+8, r.label.MinSize().Height+8)
}

func (r *myRenderer) Objects() []fyne.CanvasObject {
    return r.objects
}

func (r *myRenderer) Refresh() {
    r.label.(*canvas.Text).Text = r.w.Text
    canvas.Refresh(r.w)
}
```

### 5.2 关键规则

1. **必须调用 `ExtendBaseWidget`**：否则 `BaseWidget.impl` 为 nil，会导致数据绑定失败、事件不触发等异常
2. **`Objects()` 必须返回非空切片**：空切片或含 nil 元素的切片会导致 Widget 不可见
3. **使用指针接收者**：`canvas.Line`, `canvas.Text` 等类型方法都是指针接收者，必须用 `*canvas.Line`
4. **不要在 `Layout()` 中调用 `Refresh()`**：会导致无限递归
5. **`MinSize()` 应基于子对象计算**：不应返回固定值，否则会破坏 Grid、Adaptive 等自适应布局

### 5.3 渲染器缓存

渲染器是缓存的，每个 Widget 只有一个渲染器实例。状态变更应通过 `Refresh()` 方法传递：

```go
func (w *MyWidget) SetText(text string) {
    w.Text = text
    w.Refresh()  // 触发渲染器的 Refresh 方法
}
```

## 6. 线程安全

### 6.1 核心规则

**所有 UI 操作必须在 Fyne 主 goroutine 中执行。**

```go
// 错误示范：
go func() {
    label.SetText("Updated")  // ❌ 可能导致竞态
}()

// 正确示范：
go func() {
    result := expensiveWork()
    fyne.Do(func() {
        label.SetText(result)  // ✅ 安全
    })
}()
```

### 6.2 fyne.Do 和 fyne.DoAndWait

```go
// 非阻塞：将 fn 放入主 goroutine 执行，立即返回
fyne.Do(func() { label.SetText("async") })

// 阻塞：等待 fn 在主 goroutine 执行完毕后返回
fyne.DoAndWait(func() {
    // 需要返回值或确保执行完成的操作
    title = window.Title()
})
```

### 6.3 数据绑定是 goroutine-safe 的

```go
// 在任何 goroutine 中调用都是安全的
go func() {
    data := binding.NewString()
    data.Set("value")  // ✅ 自动通过 fyne.Do 通知监听器
}()
```

## 11. 性能优化

### 11.1 虚拟滚动

对于大数据集，必须使用虚拟滚动组件：

```go
// ✅ 虚拟列表：只渲染可见行，O(可见行数) 复杂度
list := widget.NewList(
    func() int { return 100000 },           // 总行数
    func() fyne.CanvasObject {              // 创建模板
        return widget.NewLabel("template")
    },
    func(id widget.ListItemID, obj fyne.CanvasObject) { // 更新行
        obj.(*widget.Label).SetText(fmt.Sprintf("Row %d", id))
    })

// ❌ 直接创建 100000 个 Label 放入 VBox
// 全会创建出来，导致严重性能问题
```

### 11.2 异步加载

```go
data := binding.NewStringList()

go func() {
    results := fetchLargeDataset()
    fyne.Do(func() {
        data.Set(results)
    })
}()

list := widget.NewListWithData(data, createFn, updateFn)
```

### 11.3 避免不必要的 Refresh

```go
// ❌ 每次循环都 Refresh
for _, item := range items {
    list.Add(item)
    list.Refresh()  // 浪费
}

// ✅ 批量操作后只 Refresh 一次
for _, item := range items {
    list.Add(item)
}
list.Refresh()
```
