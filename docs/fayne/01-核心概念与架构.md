# Fyne 核心概念与架构设计

## 1. 概述

Fyne 是纯 Go 语言实现的跨平台 GUI 框架，基于 OpenGL ES 2.0 实现硬件加速渲染。设计哲学核心是 **声明式 + 响应式 UI**，通过数据绑定系统实现 UI 自动更新。

### 关键特性
- **纯 Go**：零外部 GUI 依赖，编译成单一静态二进制文件
- **跨平台**：同一套 API 覆盖 Windows、macOS、Linux、iOS、Android 和 WebAssembly
- **硬件加速**：基于 GPU 的矢量图形渲染（OpenGL ES 2.0）
- **响应式数据绑定**：观察者模式的 DataItem，任何 goroutine 都可安全调用
- **内置测试支持**：`fyne.io/fyne/v2/test` 包提供完整的内存测试能力

## 2. 分层架构

Fyne 采用清晰的分层架构，各层职责明确：

```
┌───────────────────────────────────────────┐
│           应用层 (Application)             │  ← 你的代码
├───────────────────────────────────────────┤
│           组件层 (Widget)                  │  ← Button, Label, Entry 等 UI 组件
├───────────────────────────────────────────┤
│           容器层 (Container)               │  ← VBox, HBox, Grid, Border 等布局容器
├───────────────────────────────────────────┤
│           画布层 (Canvas)                  │  ← 平台无关的绘制抽象
├───────────────────────────────────────────┤
│           驱动层 (Driver)                  │  ← 平台相关实现 (glfw, mobile, wasm)
├───────────────────────────────────────────┤
│           平台层 (Platform)                │  ← OS 系统调用
└───────────────────────────────────────────┘
```

### 各层关键接口

| 层 | 关键接口 | 职责 |
|----|---------|------|
| 应用 | `fyne.App` | 生命周期管理、窗口创建、设置、偏好存储、通知 |
| 窗口 | `fyne.Window` | 窗口标题、大小、全屏、关闭、菜单栏 |
| 画布 | `fyne.Canvas` | 内容管理、缩放、聚焦、快捷键、OverlayStack |
| 基础对象 | `fyne.CanvasObject` | 所有可绘制对象的位置、大小、可见性 |
| 组件 | `fyne.Widget` / `fyne.WidgetRenderer` | 组件的声明和渲染分离 |
| 容器 | `fyne.Container` | 包含一组 CanvasObject 并提供布局 |
| 布局 | `fyne.Layout` | 自动排列子元素的位置和大小 |
| 驱动 | `fyne.Driver` | 窗口创建、事件循环、设备信息 |
| 设备 | `fyne.Device` | 设备方向、是否移动端、键盘、系统缩放 |

## 3. 核心接口详解

### 3.1 CanvasObject — 所有可绘制对象的根基

```go
type CanvasObject interface {
    MinSize() Size      // 最小尺寸
    Move(Position)      // 移动（布局管理的对象不应手动调用）
    Position() Position // 当前位置
    Resize(Size)        // 调整大小（布局管理的对象不应手动调用）
    Size() Size         // 当前大小
    Hide()              // 隐藏
    Show()              // 显示
    Visible() bool      // 是否可见
    Refresh()           // 触发重绘
}
```

### 3.2 Widget 与 WidgetRenderer — 声明与渲染分离

Fyne 中 Widget 遵循 **声明-渲染分离** 模式：

- **Widget**（声明式对象）：持有状态（文字、选中状态等），实现 `CreateRenderer()`
- **WidgetRenderer**（渲染器）：每个 Widget 实例有且仅有一个渲染器，负责实际的绘制对象

```go
type Widget interface {
    CanvasObject
    CreateRenderer() WidgetRenderer
}

type WidgetRenderer interface {
    Destroy()                          // 销毁时清理资源
    Layout(Size)                       // 接收到新尺寸时的回调
    MinSize() Size                     // 最小尺寸
    Objects() []CanvasObject           // 返回实际绘制对象列表（不能为空/nil）
    Refresh()                          // 状态变更后的重绘回调
}
```

**关键规则**：
- `CreateRenderer()` 由框架内部调用，不应在应用代码中直接调用
- `Objects()` 必须返回非空、所有元素非 nil 的切片
- `Layout()` 中不应调用 `Refresh()`
- 每个 Widget 实例内存中只有一个 Renderer

### 3.3 App — 应用程序入口

```go
type App interface {
    NewWindow(title string) Window
    Run()                // 启动事件循环（阻塞）
    Quit()               // 退出应用
    Driver() Driver
    Settings() Settings
    Preferences() Preferences
    Storage() Storage
    Lifecycle() Lifecycle
    CloudProvider() CloudProvider
    SetCloudProvider(CloudProvider)
    SendNotification(*Notification)
    Clipboard() Clipboard
    Cache() Cache
    // ...
}
```

`app.New()` 创建桌面应用，`app.NewWithID("com.example.myapp")` 创建带唯一 ID 的应用。

### 3.4 Container — 容器

```go
type Container struct {
    Layout  Layout           // 布局算法
    Objects []CanvasObject   // 子对象列表
    // 内部状态: size, position, Hidden
}
```

容器实现了 `CanvasObject`，可嵌套。推荐使用 `container.New*` 系列函数：

- `container.NewVBox(objects...)` — 垂直排列，使用子元素高度 MinSize
- `container.NewHBox(objects...)` — 水平排列，使用子元素宽度 MinSize
- `container.NewBorder(top, bottom, left, right, center...)` — 边框布局
- `container.NewGridWithColumns(cols, objects...)` — 网格布局
- `container.NewGridWithRows(rows, objects...)` — 网格布局
- `container.NewStack(objects...)` — 堆叠布局（适合叠加效果）
- `container.NewPadded(objects...)` — 带内边距
- `container.NewCenter(objects...)` — 居中
- `container.NewAdaptiveGrid(rowcols, objects...)` — 自适应网格（移动端响应方向变化）
- `container.NewWithoutLayout(objects...)` — 无布局（需手动控制大小和位置）

### 3.5 Layout — 布局算法

```go
type Layout interface {
    Layout([]CanvasObject, Size)   // 在给定 Size 内排列对象
    MinSize([]CanvasObject) Size   // 计算最小尺寸
}
```

内置 Layout 包括：
- `layout.NewHBoxLayout()` — 水平盒子
- `layout.NewVBoxLayout()` — 垂直盒子
- `layout.NewBorderLayout(top, bottom, left, right)` — 边框
- `layout.NewGridLayout(cols)` / `layout.NewGridLayoutWithColumns(cols)` / `layout.NewGridLayoutWithRows(rows)`
- `layout.NewFormLayout()` — 表单（标签-输入配对）
- `layout.NewPaddedLayout()` — 带 padding
- `layout.NewCenterLayout()` — 居中
- `layout.NewStackLayout()` — 堆叠
- `layout.NewGridWrapLayout(size)` — 网格自动换行
- `layout.NewSpacer()` — 弹性占位符

## 4. 事件系统

### 4.1 核心事件接口

```go
// 点击事件
type Tappable interface     { Tapped(*PointEvent) }
type DoubleTappable interface { DoubleTapped(*PointEvent) }
type SecondaryTappable interface { TappedSecondary(*PointEvent) }

// 键盘事件
type Focusable interface {
    FocusGained()
    FocusLost()
    TypedRune(rune)
    TypedKey(*KeyEvent)
}

// 滚动事件
type Scrollable interface { Scrolled(*ScrollEvent) }

// 拖动事件
type Draggable interface { Dragged(*DragEvent); DragEnd() }

// 快捷键事件
type Shortcutable interface { TypedShortcut(Shortcut) }
```

### 4.2 事件类型

| 类型 | 字段 | 说明 |
|------|------|------|
| `PointEvent` | `AbsolutePosition`, `Position` | 指针事件，Position 是相对触发对象左上角的坐标 |
| `KeyEvent` | `Name`, `Physical` | 键盘事件，Name 是跨平台一致的 KeyName |
| `ScrollEvent` | `PointEvent`, `Scrolled` | 滚轮事件，Scrolled 是 Delta(DX, DY) |
| `DragEvent` | `PointEvent`, `Dragged` | 拖动事件，Dragged 是 Delta(DX, DY) |

### 4.3 快捷键系统

内置快捷键（自动绑定 Ctrl+C/V/X 等）：
- `ShortcutCopy` / `ShortcutPaste` / `ShortcutCut`
- `ShortcutSelectAll`
- `ShortcutUndo` / `ShortcutRedo`

自定义快捷键通过 `Canvas.AddShortcut(shortcut, handler)` 注册。

## 5. 数据绑定系统 (Data Binding)

位于 `fyne.io/fyne/v2/data/binding` 包，提供响应式数据流。

### 5.1 核心接口

```go
type DataItem interface {
    AddListener(DataListener)
    RemoveListener(DataListener)
}
```

所有绑定 API **在任何 goroutine 中都可安全调用**，监听器回调会自动通过 `fyne.Do` 切换到主 goroutine。

### 5.2 基本绑定类型

| API | 说明 |
|-----|------|
| `binding.NewBool()` | 绑定 bool 值 |
| `binding.NewFloat()` | 绑定 float64 值 |
| `binding.NewInt()` | 绑定 int 值 |
| `binding.NewString()` | 绑定 string 值 |
| `binding.NewURI()` | 绑定 URI |
| `binding.NewUntyped()` | 绑定任意类型 (any) |

### 5.3 集合绑定类型

| API | 说明 |
|-----|------|
| `binding.NewBoolList()` | bool 列表 |
| `binding.NewFloatList()` | float64 列表 |
| `binding.NewIntList()` | int 列表 |
| `binding.NewStringList()` | string 列表 |
| `binding.NewStringTree()` | 字符串树 |
| `binding.NewStringMap()` | 字符串字典 |

### 5.4 外部绑定（绑定到现有变量）

```go
var count int
b := binding.BindInt(&count)  // 修改 count 会自动通知所有监听器
```

### 5.5 Widget 的数据绑定支持

Widget 提供 `*WithData` 构造函数实现响应式 UI：

```go
data := binding.NewString()
data.Set("Hello")

entry := widget.NewEntryWithData(data)    // 输入框变更 → data 自动更新
label := widget.NewLabelWithData(data)    // data 变更 → 标签自动更新
```

## 6. 线程模型（v2.6.0+ 重要变更）

### 6.1 核心规则

**所有 UI 操作必须在 Fyne 主 goroutine 中执行。**

```go
// 后台 goroutine 中安全更新 UI
go func() {
    // 执行耗时操作...
    result := fetchData()

    // 回到主 goroutine 更新 UI
    fyne.Do(func() {
        label.SetText(result)
    })
}()
```

- `fyne.Do(fn)` — 非阻塞地将 fn 调度到主 goroutine
- `fyne.DoAndWait(fn)` — 阻塞等待 fn 在主 goroutine 执行完成

### 6.2 迁移要求

如果使用 v2.6.0+ 版本，需要在 `FyneApp.toml` 中添加：
```toml
fyneDo = true
```

## 7. Geometry 体系

```go
type Position struct { X, Y float32 }   // 坐标
type Size struct { Width, Height float32 } // 尺寸
type Delta struct { DX, DY float32 }    // 增量
```

- `Position.Add(v)` / `Position.Subtract(v)` — 坐标运算
- `Size.Max(v)` / `Size.Min(v)` — 取最大/最小尺寸
- `Size.Add(v)` / `Size.Subtract(v)` — 尺寸运算

## 8. 主题系统

从 v2.0 开始，主题使用新的 `Theme` 接口：

```go
type Theme interface {
    Color(ThemeColorName, ThemeVariant) color.Color
    Font(TextStyle) Resource
    Icon(ThemeIconName) Resource
    Size(ThemeSizeName) float32
}
```

常用 `ThemeColorName` 常量（在 `theme` 子包中）：
- `theme.ColorNamePrimary` / `theme.ColorNameBackground` / `theme.ColorNameForeground`
- `theme.ColorNameButton` / `theme.ColorNameDisabledButton`
- `theme.ColorNameInputBackground` / `theme.ColorNamePlaceHolder`
- `theme.ColorNameScrollBar` / `theme.ColorNameHover` / `theme.ColorNameFocus`

### ThemeVariant
```go
const (
    ThemeVariantLight ThemeVariant = iota
    ThemeVariantDark
)
```

应用主题：
```go
app.Settings().SetTheme(&myTheme{})
```
